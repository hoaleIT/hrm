📊 8 TABLES STRUCTURE
═════════════════════════════════════════════════════════════════════════════════

1. USERS (Tài khoản)
   ├─ id (UUID) - Primary key from Supabase Auth
   ├─ email - Login email (unique)
   ├─ role - admin/hr/employee
   ├─ employee_id - Link to employee record
   ├─ created_at, updated_at
   └─ Row Level Security: ✅ Only see own or managed accounts

2. DEPARTMENTS (Phòng ban)
   ├─ id (UUID)
   ├─ name - Department name (unique)
   ├─ description - Department details
   ├─ manager_id - Links to users
   ├─ created_at, updated_at
   └─ Accessible by: All authenticated users

3. POSITIONS (Chức vụ)
   ├─ id (UUID)
   ├─ name - Position name (unique)
   ├─ description - Job description
   ├─ allowance - Position allowance (VND)
   ├─ created_at, updated_at
   └─ Accessible by: All authenticated users

4. EMPLOYEES (Nhân viên)
   ├─ id (UUID)
   ├─ employee_code - Unique code (NV001, NV002...)
   ├─ full_name - Full name
   ├─ email - Company email (unique)
   ├─ phone, gender, birth_date
   ├─ address
   ├─ department_id - Links to departments
   ├─ position_id - Links to positions
   ├─ salary - Base salary (VND)
   ├─ avatar_url - Profile picture URL
   ├─ status - active/inactive/on_leave
   ├─ start_date - Employment start date
   ├─ created_at, updated_at
   └─ Row Level Security: ✅ Employees see own, HR/Admin see all

5. ATTENDANCE (Chấm công)
   ├─ id (UUID)
   ├─ employee_id - Links to employees
   ├─ date - Attendance date (unique per employee)
   ├─ check_in - Check-in time
   ├─ check_out - Check-out time
   ├─ status - present/late/absent/leave
   ├─ notes - Additional notes
   ├─ created_at, updated_at
   └─ Row Level Security: ✅ Employees see own, HR/Admin see all

6. LEAVE_REQUESTS (Đơn xin nghỉ)
   ├─ id (UUID)
   ├─ employee_id - Links to employees
   ├─ leave_type - Annual/Sick/Personal/etc
   ├─ start_date, end_date
   ├─ reason - Reason for leave
   ├─ status - pending/approved/rejected
   ├─ reviewed_by - HR who reviewed (links to users)
   ├─ reviewed_at - Review timestamp
   ├─ comments - HR comments
   ├─ created_at, updated_at
   └─ Row Level Security: ✅ Employees see own, HR approve all

7. PAYROLLS (Bảng lương)
   ├─ id (UUID)
   ├─ employee_id - Links to employees
   ├─ month - Payroll month (unique per employee)
   ├─ base_salary - Base salary (VND)
   ├─ allowance - Total allowance (VND)
   ├─ bonus - Bonus amount (VND)
   ├─ deduction - Deductions (VND)
   ├─ total_salary - Total (calculated: base + allow + bonus - ded)
   ├─ status - draft/processed/paid
   ├─ created_at, updated_at
   └─ Row Level Security: ✅ Employees see own, HR/Admin manage all

8. NOTIFICATIONS (Thông báo)
   ├─ id (UUID)
   ├─ user_id - Recipient (links to users)
   ├─ type - leave_request/attendance/payroll/etc
   ├─ title - Notification title
   ├─ message - Notification message
   ├─ read - Read status (boolean)
   ├─ related_id - Related object ID
   ├─ created_at, updated_at
   └─ Row Level Security: ✅ Users see only their notifications

🔧 TROUBLESHOOTING
═════════════════════════════════════════════════════════════════════════════════

Problem: Tables don't exist
Solution: Run sql/01_create_schema.sql in Supabase SQL Editor

Problem: Can't insert data
Solution: Check RLS policies are correct, run with admin key

Problem: Can't query employees
Solution: Make sure .env.local has correct NEXT_PUBLIC_SUPABASE_URL

Problem: Slow queries
Solution: Check indexes are created, consider adding more if needed

Problem: Data not persisting
Solution: Check RLS policies allow INSERT/UPDATE/DELETE

Problem: RLS blocking access
Solution: This is intentional! Employees should only see own data


💡 BEST PRACTICES
═════════════════════════════════════════════════════════════════════════════════

✅ Always use parameterized queries (Supabase SDK handles this)
✅ Keep employee_code unique and sequential (NV001, NV002...)
✅ Use DATE type for dates, TIMESTAMP for times with timezone
✅ Store money as DECIMAL(12,2) not FLOAT
✅ Always set proper RLS policies
✅ Regular backups of Supabase data
✅ Monitor database usage in Supabase Dashboard
✅ Use indexes for frequently queried columns
✅ Keep sensitive data in RLS-protected columns
